# ソフトウェア
システムを構成するうちの一つである「ソフトウェア」について。代表的なソフトウェアの１つである「OS」の種類や役割、および、アプリケーションソフトウェアの利用方法などを解説

# オペレーティングシステム
システムは「ハードウェア」と「ソフトウェア」の２つの要素から構成される。その「ソフトウェア」は「OS」と「アプリケーションソフトウェア」の２つに分類される。
## OS（Operrating System）
* OSとは‥アプリケーションソフトウェアとハードウェアの間で処理を仲介するソフトウェア。OSの例はWindowsやmacOS、iOSやAndroid。
## アプリケーションソフトウェア
* アプリケーションソフトウェアとは‥ユーザーに特定の機能を提供するソフトウェア。例えば「カメラ機能」や「メッセージ機能」。

# OSの４つの機能
1. ジョブ管理
2. タスク管理
3. 記憶管理
4. データ管理

## １・ジョブ管理
* ジョブ管理とは‥ジョブの実行順番を管理する機能のこと。
* ジョブとは‥ユーザーがコンピュータに依頼する仕事の単位。例えば「書類の印刷」や「ファイルの複製」。
#### なぜジョブ管理が必要なのか
CPUとその他のハードウェアの処理速度が大幅に異なるから。例えばプリンターで書類の印刷をする場合、OSがなかったら印刷が終わるまでCPUは待ち状態になってしまう。
### スプーリング
* スプーリングとは‥<br>「CPU」と「低速の入出力装置」の間のデータ転送を「高速の補助記憶装置」に介入させる技術。スプーリングはプリンターの印刷時に使われる技術で、CPUとプリンターの間に記憶記録装置などを設置し、一次的にデータを保存することでプリンターの処理速度に応じてプリンターにデータを送ることでCPUは別のジョブを行うことができる。
### バッファ
* バッファとは‥<br>スプーリングで使用する補助記憶装置のように処理速度や転送速度のことなる２つの装置の差を補うことを目的として、データを一次的に保存する記憶装置のこと。装置間の差を和らげる（緩衝する）役割であることから「緩衝記憶装置」と呼ばれることもある。なおCPUと主記憶装置の処理速度の差を補う「キャッシュメモリ」や主記憶装置とハードディスクの処理速度の差を補う「ディスクメモリ」もバッファの１つ。

## ２・タスク管理
* タスク管理とは‥タスクの実行順序を管理する機能。
* タスク（プロセス）とは‥OSがCPU（ハードウェア）に出す命令の単位。
* ジョブとは‥ユーザーがコンピュータに出す命令
ユーザーはアプリケーションソフトウェアを使って写真を撮ったりする。OSはそれらの一連の処理をタスクという単位に分割してからCPUに実行させる
### マルチタスク
* マルチタスクとは‥<br>タスク管理によって複数のタスクを頻繁に素早く切り替えることによってそれらのタスクは*あたかも*同時に実行されているかのように見せる技術。CPUは１つのコアは一度に１つの作業しか実行できないので、１つのタスクが完了するまで他のタスクを待機させておく方法だと、他のタスクが長時間待たされることになってしまう。なお、マルチタスクはOSがCPUを操作することで実現している。
### タスクを切り替える方法
プリエンプティブとノンプリエンプティブに分けられる
| 種類 | 説明 |
| :---: | :---: |
| プリエンプティブ | CPUの使用権をOSが管理する方式。プリエンプティブ（preemptive）は「横取り」という意味。優先度の高いタスクが発生した場合、優先度の高いタスクからCPUの使用権を横取りして、優先度の高いタスクに割り当てる。現在の主流。|
| ノンプリエンプティブ | CPUの使用権をタスクが管理する方式。タスクの処理が完了するまで、CPUの使用権はそのタスクが持ち、処理が完了するとタスク自身がCPUの使用権をOSに戻す。このようにノンプリエンプティブではCPUの使用権をOSが管理しないため、OSの役割がよりシンプルになる。|
### タスクが生成されてから消滅するまで
タスクはその生成から消滅まで以下の３つの状態を取る
1. 実行可能状態‥いつでもタスクを実行できる状態
3. 実行状態‥タスクを実行している状態
4. 待ち状態‥タスクの入出力処理の完了を待っている状態
#### タスクの流れ
##### 1.タスクの生成→実行可能状態
タスクが生成されるとまずは実行可能状態になる
##### ２.実行可能状態→実行状態
CPUの使用権がOS、もしくはタスクに割り当てられると、タスクは実行状態に移る
##### ディスバッチ
ディスバッチとは‥OSがタスクを実行状態に移すこと
##### ３.実行状態→実行可能状態
* プリエンティブの場合
より優先度の高いタスクが発生すると、優先度の低いタスクは実行可能状態に戻り、優先度の高いタスクが実行状態に移る
* ノンプリエンプティブの場合
より優先度の高いタスクが発生すると、現在のタスクが終了するのを待ってから、優先度の高いタスクが実行状態に移る
##### 4.実行状態→待ち状態
タスクに入出力処理が必要になると、実行状態から待ち状態になる。入出力処理は時間がかかるため、OSは入出力処理が必要なタスクを待ち状態に移して、CPUに別のタスクを実行させる。
* 入出力処理とは‥CPUが入出力装置や補助記憶装置などとデータをやり取りすること
##### 5.待ち状態→実行可能状態
入出力処理が完了すると、タスクは待ち状態から実行可能状態に移る
##### 6.実行状態→タスクの消滅 
タスクの実行が完了すると、タスクは消滅する

#### タスクスケジューリング
* タスクスケジューリングとは‥実行可能状態にある複数のタスクの中から、どのタスクを実行状態に移すのかを決めること。

| 方式 | 説明 |
| :---: | :---: |
| 到着順方式 | 先に到着したタスクを先に実行する |
| 処理時間順方式 | 処理時間の短いタスクを先に実行する |
| 優先度順方式<br>（プリエンプティブの場合） | 各タスクに優先度を与えて、その優先度順に実行する。優先度の低いタスクの実行中に、優先度の高いタスクが発生したら、優先度の高いタスクに切り替える |
| 優先度順方式<br>（ノンプリエンプティブの場合） | 各タスクに優先度を与えて、その優先度順に実行する。優先度の低いタスクの実行中に、優先度の高いタスクが発生しても、現在のタスクが終了するまで切り替えない |
| ラウンドロビン方式 | タスクを一定時間ごとに切り替えて実行する |

## ３・記憶管理
* 記憶管理とは‥記憶装置を効率的に利用するための機能
#### 記憶管理の分類
「主記憶管理」と「仮想記憶管理」の二種類に分類される。基本情報技術者試験では「仮想記憶管理」が頻出。
* 仮想記憶管理とは‥<br>補助記憶装置の一部をあたかも主記憶装置であるかのように扱う方式。プログラムが主記憶装置の容量を超える大きさの場合に、そのプログラムを仮想記憶状に格納することで実行できるようになる。仮想記憶管理では、OSが「論理的な記憶領域」を作る。この「論理的の記憶領域」のことを仮想記憶という。これに対して主記憶装置の「物理的な記憶領域」のことを実記憶という。
### 仮想記憶管理の分類
「セグメンテーション方式」と「ページング方式」の二つの方式に分類される。（基本情報技術者試験ではページング方式が頻出）
* セグメンテーション方式とは‥仮想記憶と実記憶をセグメントと呼ぶ可変長の領域に分割して管理する方式。
* ページング方式とは‥<br>仮想記憶と実記憶をページと呼ぶ固定長の領域に分割して管理する方式。仮想記憶と実記憶の各ページには、先頭から順にアドレスが割り当てられ、仮想記憶のアドレスのことを「仮想アドレス」、実記憶のアドレスのことを「物理アドレス」と呼ぶ。仮想アドレスと物理アドレスは「ページアドレス」と呼ばれる表によって対応つけられます。なお、仮想記憶の各ページとページテーブルの各行は１対１で対応つけられるため、仮想記憶にあるページの総数とページテーブルの行の総数は同じになる。
### ページインとページアウト
* ページフォルトとは‥「プログラムを実行するために必要なページ」が実キオクに存在しない場合の割り込みのこと。
* ページインとは‥ページフォルトが発生すると、OSは「プログラムを実行するために必要なページ」を補助記憶装置から実記憶へ読み込む動きのこと
* ページアウトとは‥実記憶に開きページがない場合、実記憶から外すページを１つ選び、そのページを実記憶から補助記憶装置へ書き出す動作のこと
### ページの置き換えアルゴリズム
ページアウト・ページインはできるだけ発生しない方がCPUの処理能力は高くなるので、頻繁に使うページは実記憶におき、滅多に使わないページは実記憶から外すことが望ましい。どのページを実記憶から外すのかを決めるためのアルゴリズムのことを「ページの置き換えアルゴリズム」といい、三種類ある。

| 方式 | 説明 |
| :---: | :---: |
| LRU（Least Recently Used：最も最近使用されていない） | 最後に参照されてから最も長い時間が経過したページ（使われていない時間が最も長いデータ）を置き換える |
| LFU（Least Frequently Used：最も頻繁に使用されていない） | 参照頻度が最も少ないページを置き換える |
| FIFO（First In First Out：先入先出） | 読み込んでから最も長い時間が経過したページを置き換える |

### スラッシング
* スラッシングとは‥仮想記憶管理において割り当てられる実記憶の容量が小さい時に、ページアウトとページインが頻繁に起こる現象。実記憶の容量に比べて扱うデータが大きければ空きページがなくなるのでOSがページアウトとページインを頻繁に行うことになる。スラッシングが発生すると、コンピュータ全体のスループット（単位時間あたりに処理される仕事の量）が低下する

## ４・データ管理
* データ管理とは‥コンピュータが処理するデータを管理する機能。具体的には「ファイルシステム」や「バックアップ」などの機能がある。
### ファイルシステム
* ファイルシステムとは‥<br>データのまとまりを「ファイル」として扱うことでデータを管理しやすくする機能。コンピュータは「０」と「１」の２つの数値しか扱うことができないので、その数字の羅列がデータとして保存されているが、それを人間が読み取れる形でデータを「ファイル」として視覚化して人間の扱いやすい形に変換すること。
#### ファイルとディレクトリ
* ファイル（意味：書類）とは‥記憶装置に記録された情報の集まり。ファイルシステムの管理単位の１つ。
* ディレクトリ（＝フォルダ）とは‥ファイルを格納する場所。ディレクトリを活用することで、無数に存在するファイルを種類ごとに分類して格納することが可能になる。
#### ツリー構造によるデータ管理
* ツリー構造とは‥ファイルやディレクトリをファイルシステムが管理する方法。

| 用語 | 説明 |
| :---: | :---: |
| ルートディレクトリ | 階層構造の最上位のディレクトリのこと。通常はハードディスクの真下がルートディレクトリになる |
| カレントディレクトリ | ユーザが現在、作業しているディレクトリのこと |

#### 「パス」によるファイルの指定
* パスとは‥ファイルやディレクトリの場所を示す文字列のこと。
1. ディレクトリやファイルは「/」で区切る
2. パスの始まりが「/」の時は、左端にルートディレクトリが省略されているものとする（ルートディレクトリからの指定をしたパスのことを「絶対パス」という）
3. カレントディレクトリは「.」で表す（「./」のようにカレントディレクトリからの指定をしたパスのことを「相対パス」という）
4. １階層上のディレクトリは「..」で表す

### バックアップ
* バックアップとは‥通常使用している補助記憶装置とは別の補助記憶装置状にデータの予備を取っておくこと。基本情報技術者試験では、「フルバックアップ」「差分バックアップ」「増分バックアップ」の三種類が出題される
#### フルバックアップ
* フルバックアップ（完全バックアップ）とは‥常にすべてのデータをバックアップすること。メリットは、「作業の簡易さ」が挙げられる。理由は常に全てのデータがバックアップされているので、システム障害が発生しても最新のデータアップデータを使うだけでシステムを元の状態の戻すことができる。デメリットは、データ量が多い場合バックアップの作成に時間がかかること。
#### 差分バックアップ
* 差分バックアップとは‥フルバックアップ以降、新たに追加されたデータのみをバックアップすること。メリットは差分だけしかバックアップしないので、バックアップにかかる時間が短いこと。デメリットは、フルバックアップと差分バックアップの２つのデータが必要になるので復旧作業が煩雑になること。
#### 増分バックアップ
* 増分バックアップとは‥前回のバックアップから新たに追加されたデータのみをバックアップすること。メリットは前述２つのバックアップ方法よりバックアップの時間が短いこと。新たに追加された分だけバックアップするので時間はどれよりも短いが、復旧ではさらに煩雑になる。フルバックアップ＋増分バックアップした全てのデータが必要になるため。

### 開発ツール
* 開発ツールとは‥人間が書いたプログラムがコンピュータによって実行されるまでにはいくつかの変更が必要になり、その作業で使われるプログラムのこと。基本情報技術者試験では「最適化コンパイラ」「リンカ」「静的テストツール」が出題される
#### ソースコード・オブジェクトコードとコンパイラ
* ソースコードとは‥人間が読みやすいプログラム言語で書かれたプログラムのこと
* オブジェクトコードとは‥「０」と「１」で数字で羅列に変換されたプログラムのこと
* コンパイラとは‥ソースコードをオブジェクトコードへの変換を実行するツールのこと
##### コンパイラの処理実行
コンパイラが実行する処理を細かくしたとき、５つの処理を行うことでソースコードをオブジェクトコードに変換している
1. 字句解析：ソースコードを細かい要素に分けて、間違いがないかチェックする
2. 構文解析：プログラムの文法が間違っていないかチェックする
3. 意味解析：プログラムの手続きに矛盾がないかチェックする
4. 最適化：計算の順序などを改善して、効率的なプログラムに変換する
5. コード生成：オブジェクトコードのプログラムを作る
上記のように現在のコンパイラはコードを「最適化」してくれるのでそのようなコンパイラを「最適化コンパイラ」という
#### リンカとロードモジュール
コンパイラが生成したオブジェクトコードは、コンピュータが理解できる「０」と「１」の形式になっているが、まだ実行できない。なぜならこのオブジェクトコードを実行する上で必要となる別のオブジェクトコードやライブラリなどが１つにまとめられていないためである。そこで「リンカ」というツールを使って、複数のオブジェクトコードとライブラリを１つにまとめて、実行できるプログラム（ロードモジュール）を生成する

| 用語 | 解説 |
| :---: | :---: |
| リンカ | コンパイラが翻訳した複数のオブジェクトコードに、ライブラリなどを加えてコンピュータが実行できる１つのプログラムを作るツール |
| ロードモジュール | コンピュータが実行できるプログラム。実行可能プログラムともいう。ロードモジュールは「ローダ」と呼ばれるツールによって主記憶に格納（ロード）されCPUによって実行される |

#### インタプリンタ
* インタプリンタとは‥ソースコードを一分ずつ解釈しながら実行するツール

#### コンパイラとインタプリンタの違い
| 名前 | 変換方法 | メリット | デメリット |
| :---: | :---: |　 :---: | :---: |　
| インタプリンタ | １文ずつ | 書いたソースコードをすぐ実行できるので効率よくソフトウェアを開発できる | 最適化は行われないため実行時の効率が悪い |
| コンパイラ | 一括 | ソースコードの最適化が行われるため実行時の効率が良い | 一括で変換するのでインタプリンタに比べると開発の効率が悪い |

### テスト
プログラムが正しく動くかどうかはテストして確認をする。テストは大きく「人間によるテスト」と「コンピュータによるテスト」の２つに分類される。
* 人間によるテストの代表例：レビュー（複数の人が集まってドキュメントやソースコードを読み、誤りがないか確認すること）
* コンピュータによるテストの代表例：「静的テスト」「動的テスト」

| 種類 | 解説 |
| :---: | :---: |
| 静的テスト | プログラムは実行せずに、ドキュメントやソースコードをチェックすることで、プログラムの間違いを見つけるテスト手法 |
| 動的テスト | プログラムを実行し、その結果からプログラムの間違いを見つけるテスト手法 |

#### テストツール
コンピュータによるテストでは「テストツール」と呼ばれる、プログラムの間違いを見つけるプログラムを用いてテスト行う。テストツールの中でも基本情報技術者試験では「ソースコード解析ツール」がよく出題される
* ソースコード解析ツールとは‥静的テストツールの一つでソースコードを解析して、プログラムの誤りを見つけるソフトウェア

# オープンソースソフトウェア
* オープンソースとは‥ソースコードが公開されていて、誰でも自由に見たり変更したりすることができるという考え方。
* オープンソースソフトウェア（OSS）とは‥オープンソースの考え方に基づいて作られたソフトウェアのこと。オープンソフトウェアのソースコードは一般に公開されており、誰でも自由にソースコードを改変したり再配布したりすることができる
## Eclipse（エクリプス）
* Eclipse（エクリプス）とは‥OSSの統合開発環境（IDE：Integrated Development Enviroment）。Eclipseがよく使用されているのはJava言語によるシステム開発などで使用されている。
* 統合開発環境（IDE：Integrated Development Enviroment）とは‥テキストエディタやコンパイラ、テストツールなど開発に必要なツールが１つにまとまったソフトウェアのこと。
## その他のソフトウェア
* 商用ソフトウェアとは‥企業は有料で販売するソフトウェア。ソースコードは公開しない。
* パブリックドメインソフトウェアとは‥著作権が放棄されたソフトウェア。ソースコードを入手できれば自由に作り変えることが可能。
## コピーレフト
* コピーレフトとは‥プログラムの著作権は作者が保持するが、誰もがプログラムの複製、改変、再配布ができ、さらに二時著作物にもオリジナルと同じ配布条件を求める考え方。ポイントは「改変したプログラムを再配布する場合はソースコードも公開しなければならない」という点。
